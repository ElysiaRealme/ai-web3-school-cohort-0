## Minimum Path

Understand the difference between prompt, workflow, and agent by responsibility boundary:

| Type | Decision owner | Execution pattern | Best for | Main risk |
| --- | --- | --- | --- | --- |
| Prompt | Human | One-shot model output | Explanation, drafting, lightweight transformation | Hallucination and weak reproducibility |
| Workflow | Human-designed process | Fixed or mostly fixed steps | Repeatable learning tasks, WCB checks, note generation, proof packaging | Brittle exception handling |
| Agent | Model-guided planning with tools | Dynamic multi-step execution | Reading multiple sources, choosing tools, maintaining long-running state | Over-execution, state pollution, and error propagation |

## Challenge Path

Design rule for a constrained Web3 learning assistant:

Allowed:

- Read WCB task status.
- Read Handbook / Notion / public docs.
- Generate notes, checklists, concept cards, and draft scripts.
- Explain transactions, ABI, contract calls, and explorer records.
- Prepare GitHub proof material.

Forbidden without explicit human action:

- Access private keys or seed phrases.
- Sign messages.
- Approve token allowances.
- Send transactions.
- Deploy or write to contracts.
- Submit WCB write operations without preview and confirmation.

Required human checkpoints:

- Before any WCB write operation.
- Before any wallet signature.
- Before any approval.
- Before any transfer.
- Before any contract write.
- Before public proof submission if sensitive information may appear.

## Notes

The useful distinction is not "agent vs workflow" as labels. The useful distinction is where decision authority sits:

- If the human defines every step and the system only executes them, it is a workflow.
- If the model chooses which step/tool to run next based on intermediate state, it is agentic.
- If the task touches Web3 permissions, more autonomy means more safety controls, not fewer.

For this course, the Learning Agent should be agentic for reading, planning, summarizing, and proof packaging. It should remain non-agentic or human-gated for signing, approval, transfer, and contract writes.
