## 008 第一轮检查

学员回答摘要：

1. Workflow 是人为定义的固定工作流程和执行顺序，抗异常性小；Agent 是接入 AI 用于执行 Workflow 的 tool，可以自适应异常处理。
2. RAG 是从已有知识库中检索，如果知识库错了，答案也会错。
3. AI x Web3 中 Evaluation 至少要检查脚本、交易信息等。
4. Agent 可以生成交易等内容，确认需要人工。

修正与补充：

- Workflow 的关键是“人预设流程和步骤”，这个理解正确。
- Agent 不应理解为“执行 Workflow 的 tool”。更准确地说，Agent 是能根据目标、中间状态和工具返回结果，动态决定下一步动作的执行主体；Tool 是 Agent 调用的外部能力。
- RAG 的风险不只来自知识库错误，还包括检索不到正确内容、切片丢上下文、资料过期、检索结果无关、模型误读检索材料。
- AI x Web3 的 Evaluation 不只检查脚本和交易信息，还应检查来源链接、工具调用结果、网络、合约地址、函数、参数、金额、授权额度、权限影响、测试结果、人工确认记录和最终链上状态。
- 合约交互脚本可以由 Agent 生成、解释、静态检查、模拟或准备测试；但签名、授权、转账、合约部署、合约写入和 WCB 写入提交必须人工确认。
