## 013 检查题结果

结论：通过。

已确认掌握：

- EOA 由单个私钥控制，存在单点私钥风险。
- 智能账户由合约逻辑控制，可以支持限额、白名单、Session Key 有效期等可编程权限能力。
- 多签由多个 owner 和阈值规则共同控制，能降低单点私钥风险。
- 多签不能自动解决 owner 都由同一个人或同一台机器控制的问题。
- Session Key 需要限制能做什么、不能做什么、额度、有效期和是否可撤销。
- 个人测试和小额测试网活动适合 EOA；团队 treasury 适合多签；AI 助手限额操作更适合智能账户。

修正记录：

- “给 AI 助手每天最多 0.01 ETH 的操作权限”更准确地说是适合“智能账户”或“带限额规则的智能账户”，不是泛指任意智能合约。
- AI x Web3 助手不应直接控制高权限 EOA，因为 EOA 的一把私钥代表几乎全部控制权，误操作或泄露后缺少内建限额、白名单、撤销和多人审批边界。

## 015 检查题结果

结论：通过。

学员已确认掌握：

- Web3 助手需要写清允许动作和禁止动作。
- 受限 workflow 可以按 `读取用户目标 -> 生成操作草稿 -> 生成检查清单 -> 等待人工确认 -> 记录结果 -> 整理 proof` 组织。
- 查询交易状态、解码 calldata、生成 proof Markdown、检查区块浏览器 Status 属于允许动作。
- 自动签名交易、替用户授权 Token 属于禁止动作。
- 选择的 workflow 主题：测试网交易 proof 助手。
- 停止条件：网络不明确、用户要求直接签名或转账、proof 中出现敏感信息。
- 目标太大会让助手滑向自动执行高风险操作。
