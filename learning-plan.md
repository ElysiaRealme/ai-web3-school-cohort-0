## Daily Rhythm

- Morning: read WCB Learning page, choose minimum / recommended / challenge path.
- During study: write notes, evidence links, questions, and experiments.
- Evening: prepare check-in draft, submit manually on WCB or the required platform, then record the submission link or status.

## Personal Constraints

- AI foundation: basic.
- Web3 foundation: beginner.
- Programming ability: independent developer.
- Target direction: Hackathon project.
- Daily time budget: about 3 hours.
- Output language: Chinese.

## 3-Hour Daily Default

1. 45 minutes: read the weakest prerequisite topic, currently Web3 basics.
2. 60 minutes: complete one concrete practice item, preferably testnet, wallet, contract, or agent workflow related.
3. 45 minutes: convert the practice into repo proof: note, screenshot checklist, code, diagram, or README update.
4. 30 minutes: prepare WCB proof and one Handbook feedback candidate if any learning friction appeared.

## Path Levels

- Minimum path: one Handbook section, one short note, one check-in draft.
- Recommended path: section notes plus one small experiment or diagram.
- Challenge path: turn the concept into code, demo, issue, PR, or project design.
