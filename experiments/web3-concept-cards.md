## 卡片 1：Account｜账户

定义：

- Web3 账户是链上身份和资产控制入口。常见账户包括 EOA、智能账户和多签账户。

为什么重要：

- 账户决定谁能发起交易、谁能签名、谁能控制资产或合约权限。

常见误区：

- 把 Web3 账户理解成普通网站账号。普通网站账号通常由平台重置密码；Web3 账户的控制权来自私钥、智能合约规则或多签策略。

AI x Web3 连接点：

- AI 可以帮助解释账户状态和权限设计，但不应直接掌握账户根控制权。

## 卡片 2：Address｜地址

定义：

- 地址是链上账户的公开标识，可用于接收资产、查询交易记录、连接 dApp 和展示身份。

为什么重要：

- 地址是 proof、交易、合约交互和区块浏览器查询的基本入口。

常见误区：

- 认为地址公开就等于匿名。地址虽然不直接等同真实身份，但链上行为通常公开可追踪。

AI x Web3 连接点：

- Agent 可以处理公开地址和链上记录，但不应将地址和私人身份进行不必要绑定。

## 卡片 3：Wallet｜钱包

定义：

- 钱包不是存放 Token 的地方，而是管理私钥、助记词和地址，并帮助用户签名、发交易、连接 dApp 的工具。

为什么重要：

- 钱包是用户和链上系统交互的主要界面。

常见误区：

- 认为资产“在钱包里”。实际资产记录在区块链状态中，钱包只是控制和展示入口。

AI x Web3 连接点：

- Agent 可以生成操作说明和风险提示，但钱包确认动作必须由用户完成。

## 卡片 4：Private Key｜私钥

定义：

- 私钥是控制地址的核心秘密。持有私钥的人可以签名消息或交易，从而证明自己控制该地址。

为什么重要：

- 私钥泄露通常意味着对应地址资产和权限可能被他人控制。

常见误区：

- 把私钥当成普通 API key 或可恢复密码。私钥一旦泄露，无法像 Web2 密码一样简单撤回影响。

AI x Web3 连接点：

- 私钥绝不能放进 prompt、context、公开 repo、WCB proof 或 Agent 工具调用日志。

## 卡片 5：Seed Phrase｜助记词

定义：

- 助记词是一组用于恢复钱包的词，通常可以推导出多个私钥和地址。

为什么重要：

- 助记词是恢复整个钱包控制权的根秘密。

常见误区：

- 认为助记词只是“密码提示”。它实际可能控制一组地址和资产。

AI x Web3 连接点：

- 任何 AI Agent Wallet 设计都不应读取、保存、打印或上传助记词。

## 卡片 6：Signature｜签名

定义：

- 签名是用私钥对具体内容进行加密证明，表示“我控制这个地址，并确认了这段内容”。

为什么重要：

- 签名常用于登录、身份验证、订单确认、授权声明和链下承诺。

常见误区：

- 把签名当成普通登录按钮。恶意签名可能变成可执行承诺、Permit 或订单。

AI x Web3 连接点：

- Agent 可以解释签名内容和风险，但不能替用户签名。

## 卡片 7：Approval｜授权

定义：

- 授权通常是允许某个合约或 spender 使用你某类 Token 的一定额度。

为什么重要：

- 授权不是一次性付款，而是给出资产使用权限，可能长期有效。

常见误区：

- 只看当前交易金额，不看授权对象、授权额度和有效期。

AI x Web3 连接点：

- Agent 可以提醒检查 spender、Token、额度和无限授权风险，但不能替用户点击授权。

## 卡片 8：Transaction｜交易

定义：

- 交易是真正发送到链上执行的操作，例如转账、调用合约、写入状态或修改授权。

为什么重要：

- 交易会上链、消耗 Gas，并可能改变资产、权限或合约状态。

常见误区：

- 认为有交易 hash 就等于成功。交易 hash 只是查询标识，最终要看区块浏览器状态。

AI x Web3 连接点：

- Agent 可以生成交易草稿和解释参数，但不能在用户未复核时自动发送交易。

## 卡片 9：Gas

定义：

- Gas 是链上执行所需的资源费用，用来支付网络验证和执行成本。

为什么重要：

- Gas 决定链上操作成本，也会影响失败交易的损失。

常见误区：

- 把 Gas 和转账金额混为一谈。转账金额给接收方，Gas 付给网络执行者。

AI x Web3 连接点：

- Agent 可以解释 Gas、估算成本和提示异常费用，但最终交易确认仍需用户判断。

## 卡片 10：Network｜网络

定义：

- 网络是交易执行的链上环境，例如 Ethereum mainnet、Sepolia testnet、L2 网络等。

为什么重要：

- 同一个地址或合约名在不同网络上可能代表完全不同的资产和状态。

常见误区：

- 没检查当前网络就签名、授权或转账。

AI x Web3 连接点：

- Agent 生成任何交易草稿时都必须明确目标网络，不能让用户在未知网络中确认。

## 卡片 11：Testnet｜测试网

定义：

- 测试网是用于练习、开发和验证流程的网络，测试币通常没有真实经济价值。

为什么重要：

- 学习和开发应优先在测试网完成，降低真实资产风险。

常见误区：

- 认为测试网就可以随意点击。测试网也是训练主网安全习惯的地方。

AI x Web3 连接点：

- Agent 可以先在测试网帮助用户练习完整流程：生成草稿、人工复核、钱包确认、区块浏览器验证。

## 卡片 12：Transaction Hash｜交易哈希

定义：

- 交易哈希是一笔交易提交或广播后生成的查询标识。

为什么重要：

- 它是验证链上执行过程和结果的入口。

常见误区：

- 认为拿到交易哈希就表示任务完成。还需要检查 `Success / Failed`、区块高度、Gas、from / to 和调用信息。

AI x Web3 连接点：

- Agent 可以用交易哈希整理 proof，但必须记录最终状态，而不只记录 hash。

## 卡片 13：Block Explorer｜区块浏览器

定义：

- 区块浏览器是查询链上公开数据的工具，可以查看交易、地址、合约、区块和事件。

为什么重要：

- 它是 proof-of-work 和链上验证的关键工具。

常见误区：

- 只截图钱包成功提示，不去区块浏览器验证最终状态。

AI x Web3 连接点：

- Agent 可以帮助解读区块浏览器字段，并把 explorer 链接、状态、Gas 和合约地址整理成公开 proof。

## 卡片 14：Smart Contract｜智能合约

定义：

- 智能合约是部署在链上的程序，按照公开规则读取或修改链上状态。

为什么重要：

- Token、NFT、DeFi、账户抽象、多签和很多权限逻辑都依赖智能合约。

常见误区：

- 把智能合约当成普通后端 API。合约状态公开、调用可追踪、写入通常需要交易和 Gas，升级权限也需要检查。

AI x Web3 连接点：

- Agent 可以解释合约 ABI、生成调用脚本和说明读写差异；但部署和写入合约必须人工复核并确认。

## 总结

Web3 基础概念可以串成一条执行链：

```text
钱包管理助记词 / 私钥
-> 私钥控制账户和地址
-> 地址连接 dApp 或接收资产
-> 用户可能进行签名、授权或交易
-> 交易在指定网络执行并消耗 Gas
-> 得到交易哈希
-> 通过区块浏览器验证状态
-> 结果可作为公开 proof
```

AI x Web3 的安全原则：

- Agent 可以解释、生成草稿、检查风险和整理 proof。
- 私钥和助记词永远不进入 Agent context。
- 签名、授权、转账、合约写入必须人工复核和确认。
- Proof 必须能回到交易哈希、区块浏览器、GitHub commit 或 WCB 状态。
