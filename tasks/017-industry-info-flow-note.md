# 017｜行业信息流关注清单

日期：2026-05-27

状态：学习中。

## 本节目标

本节只解决一个问题：

```text
如何建立一个 AI x Web3 的长期信息流关注清单。
```

它不是让你收藏越多链接越好，而是让你有一套稳定的信息入口：

- 谁在定义问题。
- 谁在做基础设施。
- 谁在做研究。
- 谁在办活动和 hackathon。
- 谁的内容适合每天或每周跟进。

## 为什么需要信息流

AI x Web3 变化很快，如果只靠课程材料，很容易出现两个问题：

- 只知道概念，不知道行业正在做什么。
- 只看到热点，不知道哪些是长期方向。

信息流的目标不是追热点，而是形成判断力。

最小结构：

```text
官方源 -> 研究源 -> 项目源 -> 社区源 -> 个人源 -> 每周复盘
```

## 选择信息源的标准

优先级从高到低：

1. 官方文档和官方 blog。
2. 研究机构、开发者组织和生态基金会。
3. 长期建设者和研究者。
4. 高质量 newsletter / podcast。
5. 社区讨论和 X 信息流。

不要只关注短期价格、代币热点或未验证截图。

## 最小关注清单应包含什么

WCB 任务需要覆盖：

- 人物。
- 项目。
- 研究组织。
- newsletter。
- X 账号。
- repo。
- 社区。

每条记录建议包含：

```text
名称
类型
链接
为什么关注
关注频率
我准备观察什么
```

## 每周使用方法

每周只做三件事：

1. 从清单里选 3-5 个源快速扫一遍。
2. 摘录 3 条有价值的信息。
3. 选择 1 个项目或个人进入 018 做拆解。

不要把信息流变成无尽刷屏。
