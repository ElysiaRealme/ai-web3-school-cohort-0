# Week 1｜AI 向任务｜完成 AI 可交互学习产物

日期：2026-05-27

## 产物名称

测试网交易 Proof 检查器

## 主题

围绕 Week 1 的测试网交易 proof 练习，帮助用户检查一份公开 proof 是否包含关键字段，并避免把敏感信息放进公开仓库。

## 形式

静态 HTML 小工具。

## 文件路径

- `experiments/testnet-proof-checker.html`

## 使用方式

1. 在浏览器中打开 `experiments/testnet-proof-checker.html`。
2. 填写或点击“载入 Sepolia 示例”。
3. 点击“检查 Proof”。
4. 根据结果补充字段或移除敏感信息。
5. 使用生成的 Markdown 草稿整理提交材料。

## 交互点

- 输入 Network、Transaction Hash、Explorer Link、Status、From、To、Value、Gas Fee、Block。
- 勾选是否为测试网、是否人工检查、是否不含敏感信息。
- 点击检查按钮后得到“可提交 / 需要补充 / 不能提交”的反馈。
- 自动生成 Markdown proof 草稿。

## 安全边界

- 工具不连接钱包。
- 工具不发送交易。
- 工具不签名。
- 工具不授权 Token。
- 工具不提交 WCB。
- 工具只在浏览器本地检查字段完整性。
- 如果 proof 中包含助记词、私钥、API Key 或钱包备份文件，应停止提交并删除敏感信息。

## AI 生成了什么

- 页面结构草稿。
- 字段检查逻辑。
- 安全提醒文案。
- Markdown proof 生成模板。

## 我修改和确认了什么

- 选择了“测试网交易 proof 字段检查器”作为最小可交互产物。
- 使用自己的 Sepolia 测试交易作为示例数据。
- 确认可公开字段只包含交易哈希、浏览器链接、地址、状态、金额、Gas 和区块高度。
- 确认工具不处理助记词、私钥、API Key 或钱包备份文件。

## 不可靠或仍需人工判断的地方

- 工具只检查字段格式和本地输入，不会实时查询链上状态。
- `Status`、`Network`、`Explorer Link` 仍需用户在区块浏览器中人工确认。
- 工具不能判断一笔交易是否真的属于用户本人。
- 工具不能替代正式提交前的人工复核。
